# Example member / element naming (callsigns)

Use consistent names so roster lines in mission plans match how your unit addresses teams on net.

| Example name   | Typical use        |
|----------------|--------------------|
| **Charlie**    | 3rd squad / team C |
| **Sierra**     | Security / signals |
| **Alpha Kilo** | Alpha element, Kilo sub-unit |

Add the callsigns or roster IDs your group actually uses in **Required members** on the mission plan. Anyone with the operational encryption key can decrypt and read the list.

**Suggestions**

- One entry per line, or comma-separated: `Charlie, Sierra, Alpha Kilo`
- Include role if helpful: `Charlie (TL), Sierra (RTO)`

In the **Vault**, names that look like `charlie-sierra` are shown as **Charlie Sierra** with a **CS** subtitle; two- to six-letter **ALLCAPS** tokens (e.g. `CS`) stay as-is.

This file is documentation only and is not loaded by the app.
